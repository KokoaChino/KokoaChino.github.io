Lombok 代码简化框架

Lombok 是一个 Java 库，旨在通过注解简化 Java 代码的编写，尤其是在处理模型类时

它通过减少样板代码，使得开发者能够更专注于业务逻辑

基础功能

@Getter

生成所有字段的 getter 方法。使用此注解后，Lombok 会为类中的每个字段自动生成对应的 getter 方法，无需手动编写

@Setter

生成所有字段的 setter 方法。应用此注解后，Lombok 会为类中的每个字段自动生成对应的 setter 方法，无需手动编写

@AllArgsConstructor

生成一个包含所有字段的构造函数。这个注解会为类生成一个构造函数，该构造函数包含类中所有字段作为参数

@NoArgsConstructor

生成一个无参构造函数。使用此注解后，Lombok 会自动为类生成一个没有参数的构造函数

@RequiredArgsConstructor

生成一个包含所有 final 字段和带有 @NonNull 注解字段的构造函数。这个注解会为类生成一个构造函数，参数包括所有 final 字段及 @NonNull 注解的字段

@ToString

生成 toString() 方法。应用此注解后，Lombok 会为类生成一个 toString() 方法，通常包含所有字段的值

@EqualsAndHashCode

生成 equals() 和 hashCode() 方法。使用此注解后，Lombok 会为类生成 equals() 和 hashCode() 方法，基于类中的字段

@Data

生成 getter、setter、toString()、equals() 和 hashCode() 方法。此注解是 @Getter、@Setter、@ToString、@EqualsAndHashCode 的组合，适用于需要常规数据操作的类

@Value

生成不可变的类（即字段为 final 的类），包括 getter、toString()、equals() 和 hashCode() 方法。此注解会使类的字段都被声明为 final，并且自动生成构造函数

高级功能

@Builder

生成一个建造者模式的实现。应用此注解后，Lombok 会为类生成一个 Builder 类，使得创建实例时可以使用流式 API

@Cleanup

自动调用资源的 close() 方法来清理资源。使用此注解后，Lombok 会在字段的使用范围结束时自动调用 close() 方法，适用于需要清理的资源

@SneakyThrows

允许在方法中抛出检查型异常而不需要显式地声明它们。使用此注解后，Lombok 会在编译时将方法中的检查型异常包装成运行时异常，以避免手动处理

@NonNull

自动生成对字段或方法参数的非空检查。此注解会在类的构造函数、setter 方法或方法参数中自动添加空值检查，以确保字段或参数不为空

@Synchronized

生成线程安全的同步方法。使用此注解后，Lombok 会为方法生成同步代码，以确保在多线程环境下的安全执行

@Locked

用于标记锁定的类或方法。此注解的功能是确保在执行方法时对象的锁定，通常与多线程环境相关。请注意，这是一个自定义注解，不属于标准 Lombok 注解集合

@Log

生成日志记录器字段。应用此注解后，Lombok 会为类生成一个日志记录器字段，支持 java.util.logging、org.slf4j 等日志框架

实验性功能

@Accessors

定制生成的 getter 和 setter 方法的行为。使用此注解后，可以控制 getter 和 setter 方法的命名风格，比如链式调用

@ExtensionMethod

允许在类中添加扩展方法。这个注解可以让你为现有类添加新的方法，就像它们是该类的一部分一样

@FieldDefaults

配置字段的默认访问级别和其他属性。应用此注解后，Lombok 会为类中的字段设置默认的访问级别和其他属性，如是否为 final

@UtilityClass

标记一个工具类，所有字段和方法都为静态的。使用此注解后，Lombok 会自动将类中的字段和方法标记为 static，并禁止实例化该类

@StandardException

用于标记标准异常类。此注解会生成具有标准构造函数的异常类，并可配置其访问级别
