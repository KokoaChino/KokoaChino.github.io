注解

一种特殊类型的标记，可以被附加到代码的各个元素上，如类、方法、字段、参数等

它提供了一种为程序元素添加元数据的方式，这些元数据可以在编译时、类加载时或运行时被读取和处理

预设注解

在 Java 语言中定义的标准注解

  • @Override：

    用于指示一个方法覆盖超类中的方法

    编译器会检查此注解所标记的方法是否确实重写了超类中的方法，如果没有，将会抛出编译错误

  • @Deprecated：

    标记一个方法、类或字段为不推荐使用

    这意味着该元素在未来的版本中可能会被移除或不再支持

    使用这个注解的元素通常有替代方案，编译器在使用该元素时会发出警告

  • @FunctionalInterface：

    标记一个接口为函数式接口，即该接口只能有一个抽象方法

    函数式接口可以被隐式转换为 lambda 表达式或方法引用，使得代码更加简洁和可读

  • @SuppressWarnings：

    用于指示编译器忽略特定的警告信息

    它可以用来压制多种类型的警告，如未使用的变量、unchecked 类型转换等

    通过指定警告的名称，可以控制编译器在编译时的行为，以提高代码的可读性和可维护性

  • @SafeVarargs：

    用于标记一个方法的可变参数是安全的

    通常用于静态方法或实例方法中，防止出现类型安全的问题

    当使用泛型时，编译器会检查类型安全性，而使用此注解可以告知编译器该方法不会导致不安全的操作

元注解

用于描述其他注解的注解

  • @Retention：标识这个注解的生命周期，确定注解的可见性

    值 说明
    RetentionPolicy.SOURCE 注解只在源代码中存在，编译后会被丢弃
    RetentionPolicy.CLASS 注解在编译后会保留在 class 文件中，但在运行时不可用
    RetentionPolicy.RUNTIME 注解在运行时仍然可用，可以通过反射访问

  • @Target：标记注解可以应用于哪些 Java 成员，限制注解的使用范围

    值 说明
    ElementType.TYPE 可以应用于类、接口或枚举
    ElementType.FIELD 可以应用于字段
    ElementType.METHOD 可以应用于方法
    ElementType.PARAMETER 可以应用于方法参数
    ElementType.CONSTRUCTOR 可以应用于构造器
    ElementType.LOCAL_VARIABLE 可以应用于局部变量
    ElementType.ANNOTATION_TYPE 可以应用于注解类型
    ElementType.PACKAGE 可以应用于包

  • @Inherited：标记一个注解可以被子类继承。如果一个注解被标记为@Inherited，那么如果父类使用了这个注解，子类在未显式声明的情况下，也会继承这个注解

  • @Documented：标记这些注解是否包含在用户文档中

  • @Repeatable：允许同一个注解可以在同一声明上使用多次

类加载器

双亲委派机制

Java 类加载的一种机制，用于确保 Java 类加载的安全性和效率

它的核心思想是：

当一个类加载器接收到类加载请求时，它并不会自己直接去加载这个类，而是会把这个请求委派给它的父加载器去处理

只有在父加载器无法找到该类的情况下，当前加载器才会尝试加载该类

工作原理

  1. 类加载器层次结构：

    在 Java 中，类加载器是以层次结构组织的

    每个类加载器都有一个父类加载器，通常是 Java 的根类加载器（Bootstrap ClassLoader）是层次结构的最上层

  2. 加载流程：

    当一个类加载器接收到类加载请求时，它首先将请求转发给父加载器

    父加载器尝试加载这个类：

    • 如果父加载器能够加载该类，则返回这个类

    • 如果父加载器无法加载该类，当前类加载器才会尝试去加载这个类

例子

假设我们有一个类A，它被定义在用户项目中，同时也存在于 Java 核心库中

按照双亲委派机制，加载A的请求会首先被委派给根类加载器，根类加载器会查找核心库中的A类

如果找到了，加载器就会返回这个类；如果没有找到，委派给应用程序类加载器，然后才会去加载用户自定义的A类

总结

双亲委派机制是 Java 类加载的基础，确保了类加载过程的安全性和高效性

通过这一机制，Java 能够有效地管理类的加载和使用，维护 Java 应用程序的稳定性
